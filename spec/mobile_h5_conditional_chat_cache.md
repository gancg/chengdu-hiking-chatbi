# H5 按出行方式确认状态缓存会话

## 背景

用户尚未确定自驾或报团时，前期咨询属于未完成的新会话，不应在重新进入 H5 页面后继续恢复。

## 要求

- 只有用户明确选择“自驾”或“报团”，并且助手已完成该轮回答后，才允许缓存完整会话。
- 用户直接回复出行方式文字，或针对助手列出的“自驾/报团”选项回复对应数字，均视为明确选择。
- 同一句话同时出现“自驾”和“报团”但未作唯一选择，不视为已确定出行方式。
- 用户已明确选择出行方式但助手尚未回答时，不得缓存该未完成会话。
- 不满足缓存条件时，必须向浏览器状态写入空缓存；重新进入页面后视为新对话。
- 满足条件后的缓存仍沿用 12 小时、100 条消息和 512 KB 的限制。
