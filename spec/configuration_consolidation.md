# 配置集中管理

## 背景

应用运行参数已部分通过 `hiking_chatbi.config` 读取环境变量，但天气、报团链接和游侠客采集管道仍各自维护模型、地址、超时和文件路径。节假日日历也直接维护在 Python 源码中。

## 本次范围

- 所有部署及采集参数统一由 `hiking_chatbi.config` 读取。
- 提供 `.env.example`，记录可用环境变量、默认值和密钥名称。
- 报团链接模块不得重复维护站点默认地址。
- 游侠客采集器和路线管道的模型、API 地址、超时、数量及默认文件路径均可由环境变量覆盖。
- 和风天气 Host、请求超时和缓存时间由统一配置提供。
- 中国大陆节假日日历迁移至 `data/holidays.json`，代码只负责加载与查询。
- 环境变量中的整数必须明确校验；无效值应抛出包含变量名的异常。

## 暂不处理

推荐评分权重、天气预警扣分、实时交通窗口和 Agent 系统提示词属于业务策略。它们应在后续独立需求中配置化，并补充行为回归测试，避免本次整理同时改变推荐结果。

## 验收标准

- 默认配置保持现有应用行为。
- 设置对应环境变量后，配置读取结果随之变化。
- 节假日查询结果与迁移前一致。
- 源码中不再重复定义三个报团来源的默认入口地址。
- `.env.example` 不包含真实密钥。
