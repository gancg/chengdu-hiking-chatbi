# Qwen 调用稳定性与任务上下文重置

## 调用控制

- 聊天单轮最多执行 10 次模型调用，可通过 `CHATBI_QWEN_MAX_LLM_CALLS` 调整。
- 每次逻辑模型调用默认最多重试 2 次，可通过 `CHATBI_QWEN_MAX_RETRIES` 调整。
- DashScope 返回可重试错误或 SDK 网络异常时统一指数退避；中间失败记录 WARNING，耗尽
  重试后由会话层记录 ERROR。
- 路线列表工具只返回筛选和选择所需摘要；多候选推荐不向模型返回天气、完整交通与费用
  明细，减少上下文体积。单路线最终方案仍保留完整结果。
- 会话日志记录每轮实际模型调用次数和工具调用次数。

## 新任务上下文

- 路线和交通方式均已由用户明确确认时，当前任务视为完成选择阶段。
- 当前确认消息仍使用此前上下文，以便完成停车点、天气、报团等后续工具。
- 该确认之后的下一条用户消息视为新请求：发送模型前删除此前用户、助手和工具消息，
  仅保留系统消息以及从这条新用户消息开始的新任务消息。
- 新请求不得继承旧路线、交通方式、日期、天气或工具结果。
- 只有路线或交通方式之一尚未确认时继续保留上下文。
