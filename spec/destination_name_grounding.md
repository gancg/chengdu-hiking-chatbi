# 目的地参数必须以用户原话为依据

## 背景

`recommend_hiking_routes` 同时接收风景偏好和可选的 `destination_name`。模型可能把“森林”“花海”“草原”等风景偏好误填为目的地名称，而目的地检索采用模糊匹配，这些词可能命中多条路线并错误要求用户确认目的地。

## 规则

- 只有用户原话明确表达“去、爬、登、选择”等目的地意图，并提到对应地点时，才允许 `destination_name` 限定路线。
- 用户只提供出行时间、风景偏好、难度或其他普通筛选条件时，即使模型生成了 `destination_name`，推荐工具也必须忽略该参数并继续普通路线推荐。
- 被忽略的无依据 `destination_name` 不得触发“目的地名称命中多条路线”或“目的地名称未匹配”的异常。
- 用户确实点名目的地且命中多条路线时，仍应要求用户确认具体路线；不得擅自选择。
- 工具参数说明和对话策略必须明确：风景、地貌、地区偏好不得填入 `destination_name`。

## 验收

- 用户依次提供日期、森林风景偏好和适中难度，模型误传 `destination_name=森林` 时，工具不锁定路线且不抛出目的地异常。
- 用户明确说“想去巴朗山”并传入 `destination_name=巴朗山` 时，工具仍能锁定唯一路线。
- 用户明确点名一个实际命中多条路线的地点时，工具仍抛出含义明确的确认异常。
