# 游侠客报团链接在线查询

## 需求

用户明确选定路线并选择“报团”后，系统实时访问游侠客公开的一日游搜索页面，输入路线关键词并返回与所选路线相关的活动详情链接。系统不采集、展示或承诺价格、团期、余位和报名状态，也不代理后续报名操作。

## 路线检索词

路线新增可选字段 `group_tour_search_terms`，类型为字符串数组：

- 最多 5 项，每项去除首尾空白后至少 2 个字符。
- 不允许空值或重复项。
- 路线名称始终作为第一个检索词，字段内词条按配置顺序追加。
- 检索词由路线数据维护者人工审核，系统不自动学习、扩写或写回。
- 历史路线缺少该字段时按空数组处理，仅使用路线名称检索。

## 在线查询

- 使用无头 Chromium 打开游侠客一日游搜索页面。
- 搜索关键词优先使用路线的第一条人工审核检索词；没有检索词时使用路线名称。
- 在页面可见搜索框中输入关键词并提交，不再点击“成都出发”或“一日”等筛选项。
- 搜索完成后读取可见活动详情链接，并使用路线名称和全部检索词进行匹配。
- 标题匹配时忽略空白和常见标点；标题必须包含至少一个完整检索词，不使用编辑距离、大模型语义判断或区县泛化。
- 仅接受 `https` 协议、主机名为 `www.youxiake.com` 或 `m.youxiake.com`、路径为活动详情页的链接。
- 按页面顺序合并结果并按规范化 URL 去重，最多返回 5 条。
- 每项包含 `title`、`url`、`matched_term` 和 `fetched_at`。

## 错误边界

- 页面正常但没有匹配活动时返回空列表。
- 网络超时、浏览器不可用、验证码、登录限制或页面结构无法识别时，抛出含义明确的中文异常。
- 首次打开搜索页超时时允许自动重试一次；重试仍失败时抛出异常。
- 查询失败时不得回退到旧商团数据、静态报团费用或推测链接。
- 浏览器在成功或失败后都必须关闭。

## Qwen 行为

- 新增只读工具 `find_group_tour_links`，参数为已审核路线的 `route_id`。
- 只有用户已经选定路线并明确选择“报团”后才调用。
- 路线已确定且用户选择报团后直接调用，不再向用户进行任何查询前确认；
- 有结果时只展示活动标题和游侠客详情链接，并提示用户进入游侠客核实与完成后续操作。
- 空结果和在线查询失败必须使用不同提示，不得补充未经工具返回的信息。

## 部署

- Python 依赖固定为 `playwright==1.60.0`。
- 安装依赖后执行 `python -m playwright install chromium`。
- 默认页面为 `https://www.youxiake.com/search/results/0-0-0-1-0-0/azEtaTE.html`，单次超时 30 秒，最多返回 5 条。
