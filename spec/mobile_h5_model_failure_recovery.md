# H5 模型调用失败后恢复交互

## 背景

DashScope 流式调用异常最终会从 Agent 生成器抛出。Gradio 事件链因此中断，输入组件无法执行最后的解锁事件，用户看到页面持续处于不可操作状态。

## 要求

- H5 页面必须在模型流式调用最终失败时结束当前生成状态。
- 对话区域展示简短、可操作的重试提示，不向用户暴露上游异常详情。
- 输入框必须恢复可用，用户无需刷新页面即可重新提问。
- 失败轮次应以助手错误提示结束，避免历史中出现两个连续的用户消息。
- 服务端仍保留既有 DashScope 与对话调用异常日志。
- 该兜底仅作用于 H5 页面，不改变桌面 WebUI 行为。
