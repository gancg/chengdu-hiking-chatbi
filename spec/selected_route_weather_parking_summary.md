# 已选路线的天气、停车场与行程总结

## 目标

用户明确选择路线后，按照固定依赖顺序取得实时信息，并在流程末尾输出一份可直接用于出行的路线总结。

## 流程

1. 路线已明确且出发日期已确定时，先调用 `estimate_route_weather`。
2. 用户确认自驾后，仅在天气结果已取得时调用 `find_route_parking_points`。
3. 天气和停车点查询均完成后，不再重复询问路线、日期或交通方式，直接输出路线总结。
4. 停车点没有数据也视为查询完成；总结中明确写“暂无已审核停车点”，不得使用徒步起点代替。

## 执行保护

- 工具执行层必须校验依赖顺序，不能只依赖模型提示词。
- 天气结果不存在时，模型发起的停车点调用不得访问停车点服务；应返回可恢复的顺序错误，引导模型先调用天气工具。
- 路线总结所需工具尚未完成时，模型输出“接下来查询”“请稍等”等过渡文本不能结束当前轮；执行器应继续要求模型立即发起缺少的工具调用。
- 被顺序保护拦截的工具结果不算成功结果，后续仍须真正调用该工具。

## 路线总结

总结必须基于路线目录、历史推荐结果和本轮工具结果，至少包含：

- 路线名称、起终点、距离、爬升、预计徒步时长、难度、设施与风险；
- 出发日期的官方天气预警、温度和天气现象，以及工具返回的来源；
- 首选停车场名称、经纬度、停车说明和导航链接；没有数据时给出明确缺失说明；
- 自驾费用只按既有规则描述，不得把报团价格作为自驾费用。

不得根据模型知识补齐工具未返回的信息。总结完成后，本次路线规划视为结束；用户的下一条消息按新请求处理。
