# Qwen 运行时当前时间刷新

## 背景

生产环境中，容器内 `date` 命令输出正常，但用户询问当前时间时，模型回答的是上次服务器部署当天的日期。

## 问题

Qwen Agent 的系统提示在 Agent 构建时注入当前日期。生产进程长期运行时，系统提示里的日期会停留在进程启动或部署当天，导致模型在未调用工具的普通问答中使用过期时间。

## 需求

- 每次模型对话调用前，都必须按当前本地时间刷新系统提示里的当前日期上下文。
- 刷新逻辑不得依赖重启服务或重新构建容器。
- 当前日期上下文应包含本地日期、星期、年度，以及用于直接回答当前时间问题的本地时分信息。
- 不新增第三方依赖。

## 验收

- Agent 在构建后即使跨天运行，下一次 `run()` 传给模型的系统提示也应使用最新的当前日期。
- 现有日期解析、节假日和工具调用规则保持不变。
