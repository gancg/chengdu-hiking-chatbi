# 游侠客非徒步活动距离过滤

## 目标

游侠客定时流水线逐条核验候选活动时，不将徒步距离小于等于 0 的非徒步活动计入目标路线或发布到正式路线文件。

## 行为

- 单条候选完成模型生成和结构校验后，读取 `route.distance_km`。
- `distance_km` 必须是可转换为数字且严格大于 0 的值，才视为可入库徒步路线。
- `distance_km <= 0` 或无法转换为数字时，打印包含候选名称和距离的跳过信息，不加入本次完成列表。
- 被跳过候选仍应推进处理检查点，避免任务恢复后反复处理同一非徒步活动。
- 被跳过候选不占用 `--count` 数量，流水线继续读取后续候选，直到收集足量有效徒步路线。
- 跳过信息不得导致整个定时任务失败。
- 合并前应从既有正式路线文件中移除 `distance_km <= 0` 或非数字距离的历史坏数据，使已经产生错误文件的生产数据卷能够在下一次任务中恢复。

## 验收标准

- 正距离路线通过筛选。
- 零距离、负距离和非数字距离均被筛除。
- 正式发布文件中不新增被筛除的活动。
- 既有正式文件中的非徒步活动会从下一次发布结果中移除。
