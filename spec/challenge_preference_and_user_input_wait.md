# 挑战偏好直推与用户输入等待

## 背景

模型可能使用“直接回复 1 或 2 即可”要求用户选择。该句式没有问号，也不包含“请回复”，旧检测会误认为无需等待并继续调用工具。另外，用户已经说明“难一些”“挑战”或“高难度”时，系统不应重复询问强度。

## 规则

- 助手输出“直接回复”时，当前轮必须停止，不得执行同一模型输出中的工具调用，也不得继续下一次模型调用。
- 等待判断应跨流式分片保留；问题文本和工具调用位于不同分片时仍须拦截工具。
- 用户已提供未来日期、明确挑战偏好并请求推荐时，信息已经足够，应直接推荐，不再询问强度、体力、距离、爬升或难度。
- “难一些”“难一点”“挑战”“高难度”“难度较高”映射为 `max_difficulty=hard`。
- “不要太难”“不想太难”“不能太难”等否定表达不得识别为挑战偏好。

## 验收

“下周五想去徒步，想选难一些的路线，有推荐吗”完成日期解析后，应直接以 `max_difficulty=hard` 调用推荐工具。若助手已经输出“直接回复 1 或 2 即可”，则必须等待用户回复，不能继续推荐。
