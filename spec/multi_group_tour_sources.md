# 多商团报团链接聚合

## 需求

`find_group_tour_links` 应支持同时查询多个成都周边商团。首批来源为游侠客、大鹅和蜜多，暴走村不再接入。

- 大鹅入口：`https://www.cddee.cn/`
- 蜜多入口：`https://cdmdtb.360jlb.cn/events?mid=52240`

## 聚合规则

- 每个商团使用独立 Provider，聚合层不包含网站专属的抓取和链接校验规则。
- 查询结果增加 `source_name`，用于明确链接所属商团。
- 按 Provider 配置顺序合并结果，并按规范化后的 URL 去重。
- 单个来源失败时继续查询其他来源；至少一个来源查询成功时返回成功来源的结果。
- 所有已启用来源均失败时，抛出含义明确的中文异常。
- 未配置网站地址的来源不启用，不发起网络请求。

## 来源规则

- 大鹅使用主城区公开活动列表的 `title` 参数搜索，详情链接仅接受 `www.cddee.cn/News_read_id_<数字>.shtml`。
- 蜜多读取 `mid=52240` 的公开活动列表，并在本地按审核检索词过滤；详情链接仅接受同一主机的 `/event?id=<数字>&mid=52240`。
- 两站均只读取标题和详情链接，不读取或返回价格、团期、余位及报名状态。

## 待补信息

如商团改版导致列表或详情链接结构变化，应先更新本 spec 和中文测试，再调整对应 Provider。
