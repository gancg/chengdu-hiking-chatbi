# 自驾停车点与导航

## 目标

当用户明确选择自驾时，路线推荐应返回经过人工审核的停车点及可点击的导航链接，
帮助用户导航至停车位置，而不是把徒步起点坐标误当作停车位置。

## 数据模型

新增 `route_parking_points` 表，每条路线可维护多个停车点：

- `route_id`：关联路线，路线删除时级联删除停车点。
- `name`：停车点名称，同一路线内不得重复。
- `latitude`、`longitude`：GCJ-02 坐标，分别限制在 `[-90, 90]`、`[-180, 180]`。
- `note`：停车限制、收费、从停车点步行至徒步起点等说明，可为空。
- `is_recommended`：是否为该路线首选停车点；同一路线最多一个首选点。
- `is_reviewed`：是否经过人工审核；只有审核点可对用户展示。
- `source_url`、`updated_at`：来源和 ISO 8601 更新时间。

现有 `routes.parking` 继续表示停车概况，不承担导航定位职责。

## 导入与读取

- 路线导入项可选包含根级 `parking_points` 数组；缺省时兼容历史数据，并保留数据库
  已有停车点。
- 明确提供 `parking_points` 时，完整替换该路线原有停车点，空数组表示清空。
- 停车点字段、坐标、布尔值、时间、重复名称和首选点数量必须在写库前校验。
- 路线读取结果附带 `parking_points`，仅包含已审核停车点，首选点排在最前。

## 推荐与导航

- 仅当请求的 `transport_modes` 明确包含 `self_drive` 时，推荐结果包含
  `parking_points`；未指定交通方式或选择其他方式时不推断用户自驾。
- 每个停车点返回原始结构化信息，并生成高德地图与百度地图 HTTPS 导航链接。
- 没有已审核停车点时返回空数组，不使用徒步起点坐标兜底，也不虚构停车位置。
- Qwen 面向自驾用户展示停车点名称、导航链接和停车说明，并提醒以现场管制为准。
