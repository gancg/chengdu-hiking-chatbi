# 独立移动端 H5 ChatBI

## 目标

在保留现有 WebUI 页面和 7860 端口行为不变的前提下，新增独立的手机 H5 对话页面。H5 默认监听 7861，与 WebUI 共用业务 Agent、数据和查询工具，但使用独立组件树、样式和浏览器会话。

## 页面

- H5 顶部使用“ChatBI助手”短标题，右上角以 64px 图标展示现有徒步助手头像，并与左侧标题、副标题垂直居中；其下展示聊天区、前三条快捷问题和文本输入框。
- H5 不创建助手介绍侧栏或插件列表，但后台工具调用能力保持不变。
- 快捷问题采用纵向列表，每个问题独占一行，并在用户对话后继续保留在输入框下方。
- 输入区固定于底部安全区域之上，聊天区使用动态视口高度，长内容不得造成横向滚动。
- H5 使用独立 CSS 和页面初始化脚本移除整行 Gradio 页脚；现有 WebUI 组件树和 `appBot.css` 不变。

## 启动与配置

- `CHATBI_H5_HOST` 默认 `127.0.0.1`，`CHATBI_H5_PORT` 默认 `7861`。
- `qwen-h5` 只启动 H5；`qwen-web` 仍只启动现有 WebUI。
- `app` 同时启动 API、WebUI 和 H5。WebUI 非阻塞启动、H5 在前台运行，退出或启动失败时关闭已启动的页面和 API。
- Docker 和 Compose 同时开放 8000、7860、7861。

## 验收

- WebUI 原有测试和页面样式保持通过。
- H5 页面源码不包含插件列表或助手侧栏创建逻辑，独立样式隐藏页脚并包含动态视口、安全区域和横向溢出保护。
- 组合启动使用配置的三个端口，并可靠清理服务。
- 不新增第三方依赖，不改变模型回复格式。
