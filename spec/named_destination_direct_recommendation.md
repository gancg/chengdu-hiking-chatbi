# 点名目的地后直接推荐

## 背景

用户已经明确说出想去的地方，并且已审核路线库中存在对应路线时，目的地本身已经足以限定候选范围。继续询问体力、距离、爬升、难度、人数或设施偏好，会让对话显得机械。

## 规则

- Agent 使用已审核路线的名称和人工审核检索词判断用户点名的目的地是否有库内路线。
- 命中库内路线但尚未确定具体出发日期时，只确认出发日期，不询问其他路线偏好或交通方式。
- 出发日期确定后，立即查询日期类型并推荐命中的路线，不再询问人数、体力、经验、距离、爬升、难度、风景、设施或最晚返回时间。
- 推荐仍须遵守既有天气、风险、单日路线和工具数据真实性规则。
- 未命中库内路线时，继续使用原有引导式需求访谈。

## 验收

- “想去巴朗山”命中样例库的巴朗山路线，并进入只确认日期的策略。
- 后续用户给出明确日期时，策略要求立即推荐巴朗山路线且禁止继续收集其他偏好。
- 普通的“想找一条徒步路线”不触发点名目的地策略。
