# 天气预警查询时段规范

## 目标

为避免仅按预计徒步耗时查询天气预警而遗漏早晚风险，路线天气查询和路线推荐统一使用更保守的固定时段。

## 规则

- 预警判断时段固定为出发日期当天 `08:30–19:00`。
- 时段沿用 `departure_at` 的时区信息。
- 独立天气查询与路线推荐必须使用相同的预警判断时段。
- 现有返回字段 `hiking_start_at`、`hiking_end_at` 保持兼容，但其值表示用于天气预警判断的安全覆盖时段。

## 验收标准

- 任意出发时间查询天气时，返回的预警判断时段均为当天 `08:30–19:00`。
- 路线预计徒步结束后、但在当天 `19:00` 前生效的官方预警仍会影响推荐结果。
