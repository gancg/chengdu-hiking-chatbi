# 法定节假日查询

## 背景

路线推荐和交通估算使用 `is_holiday` 区分普通日期与节假日，但 Qwen 当前只能
依靠模型记忆判断节日日期，可能给出错误日期并错误影响交通估算。

## 需求

- 增加只读节假日查询工具 `resolve_public_holiday`。
- 支持按节日名称查询假期日期，也支持按 ISO 日期判断是否处于假期。
- 按日期查询时，同时返回可核验的星期名称和交通日期类型：工作日、周末或节假日。
- Qwen 遇到“端午节”“国庆”等节日表达时，必须先调用工具，不得自行推断日期。
- Qwen 在确定出发日期后，必须按具体日期调用工具，不得自行计算星期或交通日期类型。
- Qwen 仅能根据工具结果设置路线、交通和天气工具的 `is_holiday`。
- 未收录年份或日期时返回明确的未知结果，不得猜测。
- Qwen 系统提示应包含本地已收录节假日摘要，作为防止模型凭记忆编造节假日日期的兜底上下文。
- Qwen 不得输出未由节假日工具或已收录节假日摘要支持的具体节假日日期。

## 数据范围

首版使用本地可审计的中国大陆全国性节假日日历，覆盖 2026 年。工具结果应明确
返回数据来源，后续可在不改变工具契约的情况下替换为官方数据源适配器。

## 验收

- 查询“端午节”和年份 `2026` 返回端午节假期及节日当天。
- 查询 `2026-06-20` 返回处于端午节假期。
- 查询 `2026-06-15` 返回星期一和工作日。
- 查询未收录年份返回未知结果。
- Qwen Agent 注册节假日查询工具，并包含禁止自行推断节假日日期的规则。
- Qwen Agent 系统提示包含 `2026` 年端午节当天 `2026-06-19` 和假期 `2026-06-19` 至 `2026-06-21`。
