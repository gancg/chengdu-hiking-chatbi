# 多路线候选天气展示

## 问题

多条候选路线的起点坐标可能不同。使用其中一条路线的天气作为所有候选路线的统一天气参考，会让用户误以为该天气适用于全部路线。

## 规则

- 推荐结果只有一条路线时，可以展示该路线起点坐标对应的天气信息。
- 推荐结果包含多条路线时，不展示天气参考，也不把任意一条路线的天气概括为候选路线的共同天气。
- 用户选定具体路线且已有具体出发日期后，应立即调用该路线的天气查询工具并展示天气，不需要等待用户先选择交通方式。天气展示完成后再询问自驾或报团。
- 如果用户确认路线时尚未明确出行方式，天气展示后必须使用固定的自驾/报团选项再次询问；如果用户已经明确选择自驾或报团，则沿用该选择，不得重复询问。
- 用户已选定路线、已有具体出发日期并确认自驾时，如果当前对话尚无该阶段的天气查询结果，必须先调用 `estimate_route_weather`，不得用“出发前自行查看天气”等泛化提醒代替工具查询。
- 推荐服务仍可在内部使用逐路线天气进行安全过滤和评分，但自然语言推荐工具向模型返回多条候选时应移除天气详情，避免误用。
- Agent 执行层收到多条推荐结果后，只允许模型展示候选路线和请求用户选择，随后必须结束当前运行并等待新的用户消息。同一用户轮次中模型继续生成的天气、交通或报团工具调用均不得执行。
