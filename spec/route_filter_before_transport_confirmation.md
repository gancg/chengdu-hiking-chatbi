# 选定路线后确认交通方式

## 背景

前期需求访谈的目标是先理解用户何时出发、期待什么样的徒步路线。交通方式过早介入会打断路线偏好表达，也会在候选路线尚未形成时增加不必要的选择。

## 规则

- 前期对话先确认具体出发时间，再通过距离、爬升、难度、体力或经验、风景和设施等条件筛选符合用户预期的候选路线。
- 前期对话不得主动询问或要求用户选择交通方式。
- 形成候选路线时，允许不传 `transport_modes`，不得为了调用推荐工具而虚构交通方式。
- 给出候选路线后，下一步必须先让用户明确选择其中一条路线，不得在同一轮追加交通方式问题。
- 用户尚未明确选定路线时，不得询问交通方式，也不得展示交通方式选项。
- 用户明确选定某条路线后，再单独确认交通方式，选项为自驾、报团、公共交通。
- 如果用户提前主动提供交通方式，可以暂存该信息，但仍须先完成路线选择；不得因此改变“先选路线、后确认交通方式”的对话节奏。

## 验收

- Qwen Agent 系统提示明确前期只确认出发时间和路线筛选条件，不询问交通方式。
- 早期访谈引导不展示自驾、公共交通、报团等交通方式选项。
- 推荐候选路线的同一轮只引导用户选择一条路线，不同时询问交通方式。
- 用户明确选择路线后，下一轮才询问自驾、报团或公共交通。
- 路线推荐仍可在未指定 `transport_modes` 时返回候选路线。
