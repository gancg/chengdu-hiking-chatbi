# 游侠客定时采集诊断日志

## 目标

完善游侠客路线定时抓取、筛选、生成、发布和数据库导入全链路日志，使生产问题能够通过候选名称、来源 URL、拒绝原因和阶段结果定位。

## 行为

- 流水线使用标准 `logging`，遵循 `CHATBI_LOG_LEVEL`，不再用 `print` 输出运行状态。
- 启动时记录目标数量、列表 URL、候选检查点、进度检查点和输出文件路径。
- 列表抓取逐页记录实际 URL、页面候选数、规范化活动数以及停止分页的原因。
- 详情抓取前记录候选序号、产品标题和详情 URL；完成后记录正文字符数和网页提取字段，不记录完整正文。
- 资格筛选分别说明缺少徒步特征、未确认成都出发、未确认单日行程和检测到多日行程等原因。
- 模型生成逐次记录尝试次数；校验失败记录 URL 和具体校验异常，最终失败保留原异常链。
- 距离不合格、身份重复和历史坏数据清理日志必须包含名称、路线 ID、URL 和关键字段。
- 发布前后记录既有、本次、合并后路线数量及两个输出路径。
- 调度器记录实际子进程命令、工作目录、退出码，以及数据库校验导入的源路径、目标路径和数量。
- 日志不得包含 API Key、Authorization 头、完整详情正文或完整模型响应。

## 验收标准

- 资格判断能够返回具体且可组合的拒绝原因。
- 模型校验重试日志包含路线名称、URL、次数和错误。
- `youxiake_route_pipeline.py` 中不存在 `print` 状态输出。
- 现有路线流水线与调度器测试保持通过。
