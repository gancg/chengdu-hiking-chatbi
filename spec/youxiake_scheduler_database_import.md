# 游侠客调度结果自动导入数据库

## 目标

游侠客每日调度器成功更新 `sample_routes.json` 后，自动校验完整文件并将路线增量导入 `chatbi.db`，使正在运行的应用能够读取最新路线。

## 行为

- 只有路线流水线返回成功状态后才执行数据库导入。
- 数据文件路径使用 `CHATBI_SAMPLE_DATA_PATH`，数据库路径使用 `CHATBI_DB_PATH`。
- 导入前必须读取完整 JSON，确认根节点是数组，并逐条执行现有路线结构校验。
- 必须在所有路线都通过校验后才开始修改数据库；不得导入部分有效数据。
- 数据库导入使用更新/插入语义，不删除数据库中仅存的路线及关联反馈。
- 流水线失败、文件读取失败、JSON 解析失败、结构校验失败或数据库导入失败时，本次任务返回失败并记录异常，调度器等待下一周期重试。
- 导入成功日志应包含路线数量、数据文件路径和数据库路径。

## 验收标准

- 流水线成功后数据库导入恰好执行一次。
- 流水线失败时不触碰数据库。
- 不完整的 `sample_routes.json` 不会创建或修改目标数据库。
