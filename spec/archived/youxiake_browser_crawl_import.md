# 游侠客浏览器抓取入库

## 需求

游侠客列表页和详情页依赖前端渲染时，爬虫应复用已验证的浏览器抓取逻辑获取活动列表、详情文本和页面结构，再按现有商团入库规则写入数据库。

## 行为

- 未传入测试 fetcher 时，游侠客 provider 默认使用 Playwright 打开列表页和详情页。
- 列表页从 `div.linesSearchTitle` 下的链接发现活动 URL，并继续使用配置中的关键词过滤。
- 详情页抽取产品名称、价格、日历中的未来团期、集合点或出发地、费用包含说明。
- 若页面没有精确集合点但有出发地，可写入“出发地（集合点待人工确认）”，产品仍保持 `reviewed=false`。
- 若费用包含等商团必填字段仍无法抽取，保存到 `crawl_candidates`，不得写入业务产品表。
- 入库产品、路线候选、审计事件和下架删除仍复用既有 `tour_crawler` 流程。

## 边界

- Playwright 未安装或浏览器不可用时，抛出明确异常。
- 不绕过登录、验证码或其他访问限制。
- 浏览器抓取只解决公开页面的动态渲染，不改变待审核数据默认不进入推荐结果的规则。
