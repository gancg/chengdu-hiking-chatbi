# 单路线可靠网页采集工具（Route 第一阶段）

## 目标与边界

提供一次采集一条路线的 CLI。工具根据路线名称搜索游侠客、两步路和小红书
公开网页，在不登录、不复用 Cookie、不绕过验证码的前提下提取 `route` 字段，
生成阶段性 JSON 草稿和证据报告。工具不写数据库，不处理 `costs` 和 `traffic`。

## 命令和输出

```text
python -m hiking_chatbi collect-route --name <路线名称> --output-dir <目录>
```

输出 `<slug>.route.json`、`<slug>.evidence.json` 和
`<slug>.evidence.md`。route 文件顶层仅包含 `{"route": {...}}`；未知标量为
`null`，未知数组为空数组，`reviewed` 固定为 `false`。

## 搜索、消歧和抽取

- 每个平台使用独立 Provider，通过公开搜索页最多发现并检查前 5 条候选。
- 单个平台失败只写入证据报告。登录页、验证码、空正文和不支持的 URL 不重试绕过。
- 从起终点、路线类型和关键途经点构造走法指纹并聚类；选择获得独立平台数最多
  的路线簇，并列时选择各平台排名总和最小的簇。
- 两步路的结构化距离、耗时、最高海拔、爬升和起终点优先使用确定性规则。
- 其他自然语言由 Qwen 产生字段候选、归一值和原文引用。引用无法在来源可见正文
  中定位时作废。缺少密钥或模型失败时保留规则结果并记录失败。

## 可靠性规则

- 同一平台的多个页面只算一个来源。字段至少由两个平台一致支持才写入 route JSON；
  单源、冲突和过期值只进入证据报告。
- 距离容差为 `max(10%, 0.5km)`，爬升为 `max(15%, 100m)`，最高海拔为
  `100m`，耗时为 `max(20%, 30分钟)`。其他数值须完全一致。
- 文本由 Qwen 给出共同归一值；本地必须确认两个来源的原文引用存在，且否定关系一致。
- 停车、补给、厕所、信号、露营和风险仅允许发布日期在采集日前 12 个月内的
  候选参与共识；无发布日期只保留在报告。轨迹硬指标不设固定年限。
- 两平台一致字段为 `0.92`，三平台一致为 `0.96`。`route.confidence` 取全部
  非空采集字段的最低分；没有合格字段时为 `null`。
- `id`、`collected_at`、`updated_at`、`reviewed` 为确定性元数据，不参与最低分。
  `source_url` 使用选中路线簇排名最靠前的页面，`source_name` 列出实际提供合格
  证据的平台。

## 失败与审计

证据 JSON 保存全部搜索候选、选中路线簇、逐字段候选、平台、URL、原文、页面日期、
归一值、置信度、拒绝原因、冲突和 Provider/Qwen 错误。任何异常必须包含明确中文
上下文；不得使用默认值掩盖失败。
